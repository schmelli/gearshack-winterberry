# Implementation Plan: Integrated Image Search

**Branch**: `030-integrated-image-search` | **Date**: 2025-12-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/030-integrated-image-search/spec.md`

## Summary

Replace the placeholder "Image Search coming in V2" popover in MediaSection with a fully functional image search feature. Users can click the search button, view a 3x3 grid of product images from Serper.dev, and click to select an image that populates the form field. The search is proxied through a Next.js Server Action to keep the API key secure.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode) + Next.js 16+ (App Router), React 19+
**Primary Dependencies**: Next.js Server Actions, Serper.dev Images API, shadcn/ui (Button, AspectRatio), react-hook-form
**Storage**: N/A (no persistence - just API calls and form field population)
**Testing**: Manual testing (lint, build validation)
**Target Platform**: Web (desktop and mobile browsers)
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Search results within 3 seconds (SC-002)
**Constraints**: API key must never be exposed to client (FR-011, FR-012)
**Scale/Scope**: Single component update + new Server Action
