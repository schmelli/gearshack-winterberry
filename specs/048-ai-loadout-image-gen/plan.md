# Implementation Plan: AI-Powered Loadout Image Generation

**Branch**: `048-ai-loadout-image-gen` | **Date**: 2025-12-14 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/048-ai-loadout-image-gen/spec.md`

## Summary

Implement AI-powered image generation for GearShack loadouts using Cloudinary AI. When users create or edit loadouts, they can generate contextual outdoor photography-quality hero images by clicking a "Generate Image" button. The system analyzes the loadout's title, description, and season to construct appropriate prompts and generate high-resolution images (16:9 aspect ratio) optimized for both mobile and desktop. The feature includes regeneration capabilities (up to 3 variations stored per loadout), automatic retry logic for transient failures, silent fallback to curated defaults when AI services are unavailable, and adaptive text overlay contrast enforcement for accessibility compliance.

**Technical Approach**: Extend existing Cloudinary infrastructure to integrate Cloudinary AI's generative capabilities, following the project's Feature-Sliced Light architecture with custom hooks for business logic (`useLoadoutImageGeneration`) and stateless UI components. Leverage existing `next-cloudinary` integration patterns and Supabase storage for metadata persistence.

## Phase 1: Design & Contracts

**Prerequisites**: `research.md` complete with all NEEDS CLARIFICATION resolved

### 1. Data Model Design

**Output**: `data-model.md`

#### Entities

**GeneratedLoadoutImage** (new table: `generated_images`)
- `id` (UUID, primary key)
- `loadout_id` (UUID, foreign key to `loadouts.id`, ON DELETE CASCADE)
- `cloudinary_public_id` (string, unique, indexed)
- `cloudinary_url` (string, secure HTTPS URL)
- `prompt_used` (text, the constructed generation prompt)
- `style_preferences` (JSONB, nullable, stores user-selected style options)
- `generation_timestamp` (timestamptz, indexed for history ordering)
- `alt_text` (text, nullable, descriptive text for accessibility)
- `is_active` (boolean, marks currently selected image for loadout)
- `created_at` (timestamptz)

**Loadout** (extend existing table)
- Add column: `hero_image_id` (UUID, nullable, foreign key to `generated_images.id`, ON DELETE SET NULL)
- Add column: `image_source_preference` (enum: 'ai_generated' | 'manual_upload' | null)

#### Relationships

- One Loadout has many GeneratedLoadoutImage (1:N)
- One Loadout has one active hero_image (1:1 via hero_image_id)
- GeneratedLoadoutImage history per loadout limited to 3 (enforced by application logic with auto-deletion of oldest)

#### Validation Rules

- Loadout MUST have at least name and season to generate images
- GeneratedLoadoutImage.prompt_used MUST NOT be empty
- Maximum 3 GeneratedLoadoutImage records per loadout_id (soft delete oldest when inserting 4th)
- cloudinary_url MUST match pattern: `https://res.cloudinary.com/{cloud_name}/...`

#### State Transitions

```
[No Image]
  → (user clicks "Generate Image") → [Generating]
  → (API success) → [Success + Active Image]
  → (API failure) → [Retrying]
  → (retry failure) → [Fallback Applied]

[Active Image]
  → (user clicks "Generate Another") → [Generating New Variation]
  → (new success) → [Multiple Variations Available]
  → (user selects previous) → [Switch Active Image]
```

### 2. API Contracts

**Output**: `contracts/` directory with TypeScript/Zod schemas

#### Contract 1: Image Generation Request

**File**: `contracts/image-generation-request.schema.ts`

```typescript
import { z } from 'zod';

export const ImageGenerationRequestSchema = z.object({
  loadoutId: z.string().uuid(),
  title: z.string().min(1).max(200),
  description: z.string().optional(),
  season: z.enum(['spring', 'summer', 'fall', 'winter']),
  activityTypes: z.array(z.enum(['hiking', 'camping', 'climbing', 'skiing', 'backpacking'])).optional(),
  stylePreferences: z.object({
    template: z.enum(['cinematic', 'documentary', 'magazine', 'instagram']).optional(),
    timeOfDay: z.enum(['golden_hour', 'blue_hour', 'midday', 'dawn', 'dusk']).optional(),
    atmosphere: z.string().max(50).optional(), // "misty morning", "dramatic lighting"
  }).optional(),
});

export type ImageGenerationRequest = z.infer<typeof ImageGenerationRequestSchema>;
```

#### Contract 2: Image Generation History

**File**: `contracts/image-generation-history.schema.ts`

```typescript
import { z } from 'zod';

export const GeneratedImageSchema = z.object({
  id: z.string().uuid(),
  loadoutId: z.string().uuid(),
  cloudinaryPublicId: z.string(),
  cloudinaryUrl: z.string().url(),
  promptUsed: z.string(),
  stylePreferences: z.object({
    template: z.string().optional(),
    timeOfDay: z.string().optional(),
    atmosphere: z.string().optional(),
  }).nullable(),
  generationTimestamp: z.date(),
  altText: z.string().nullable(),
  isActive: z.boolean(),
  createdAt: z.date(),
});

export type GeneratedImage = z.infer<typeof GeneratedImageSchema>;

export const ImageHistoryResponseSchema = z.object({
  loadoutId: z.string().uuid(),
  images: z.array(GeneratedImageSchema).max(3), // Maximum 3 per loadout
  activeImageId: z.string().uuid().nullable(),
});

export type ImageHistoryResponse = z.infer<typeof ImageHistoryResponseSchema>;
```

#### Contract 3: Cloudinary AI Generation API

**File**: `contracts/cloudinary-ai-generation.ts`

```typescript
import { z } from 'zod';

export const CloudinaryAIGenerationRequestSchema = z.object({

export type CloudinaryAIGenerationRequest = z.infer<typeof CloudinaryAIGenerationRequestSchema>;

  public_id: z.string(),
  width: z.number(),
  height: z.number(),
  format: z.string(),
  resource_type: z.literal('image'),
  created_at: z.string(),
});

export type CloudinaryAIGenerationResponse = z.infer<typeof CloudinaryAIGenerationResponseSchema>;
```
