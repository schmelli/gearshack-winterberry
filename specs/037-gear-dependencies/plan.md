# Implementation Plan: Smart Gear Dependencies

**Branch**: `037-gear-dependencies` | **Date**: 2025-12-09 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/037-gear-dependencies/spec.md`

## Summary

Implement parent/child relationships between gear items so that when a user adds a main item (e.g., Packraft) to a loadout, the system detects and prompts for linked accessories (e.g., Paddle, PFD). The feature uses a `dependencyIds` array on GearItem for storage, a new Dependencies tab in the Gear Editor for link management, and a modal dialog in the Loadout Builder for dependency prompts.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Framework**: Next.js 16+ (App Router), React 19+
**Primary Dependencies**: shadcn/ui, Tailwind CSS 4, zustand, react-hook-form, zod, sonner (toast), lucide-react
**Storage**: Firebase Firestore (`userBase/{uid}/gearInventory`)
**Testing**: Manual testing (no automated test suite currently)
**Target Platform**: Web browser (responsive, mobile-first)
**Project Type**: Web application (Next.js monorepo)
**Performance Goals**: Dependency resolution <100ms for typical inventory sizes (<1000 items)
**Constraints**: Client-side only (no server-side processing), offline-capable via localStorage
**Scale/Scope**: Single user inventory, typical size 50-500 items, 1-10 dependencies per item
